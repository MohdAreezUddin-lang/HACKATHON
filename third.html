File Path System

Types of File Paths:

* Relative Path:

This is like giving directions from your current location. It describes the location of a file relative to the current HTML file.

* Absolute Path:

This is the complete path to a file, starting from the root directory.

Why Internal Absolute Paths are Avoided?

Using absolute paths to local files is blocked by browsers for security reasons, as it could give unauthorized access to a user's data.

HTML Boilerplate Code

* Necessity of Boilerplate:

The boilerplate code is important even though HTML can render content without it.

* HTML's Error Tolerance:

HTML is designed to be error-tolerant, meaning it will try to render content even if there are errors in the code.

* Browser's Role in Rendering:

Browsers automatically add missing HTML structure (like (html), (head), and (body) tags) if they are not present, which can cause unexpected styling issues.

Breakdown of Boilerplate Elements:

* (!DOCTYPE html):

Specifies that the document is an HTML5 document.

* (html lang="en"):

Defines the root of the HTML document and specifies the language of the content.

* (head):

Contains metadata about the HTML document that is not displayed on the page.

* (body):

Contains all the visible content of the HTML document.

DIV, Class, and ID-

* DIV (Division):

A generic container element used to group other HTML elements together.

* Class:

An attribute used to group multiple HTML elements, allowing them to share the same styling or JavaScript functionality.

* ID:

A unique identifier for a single HTML element.

Web Pages (Header, Main, Footer)-

* Header:

Contains introductory content, navigation links, and branding elements.

* Main:

Encompasses the primary content unique to each page.

* Footer:

Typically contains copyright information, contact details, and other supplementary content.
